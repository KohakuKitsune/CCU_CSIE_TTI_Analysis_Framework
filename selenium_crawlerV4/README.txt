這是 python selenium_crawlerV4 的 README

(1)程式功能: selenium_crawlerV3　模擬自動化操作瀏覽網頁並下載道路資料
(2)使用方法: 在input.txt輸入參數後，直接執行python .\selenium_crawlerV4.py(所需檔案都預設要在同一個目錄)
(3)執行環境: windows, python 3.10.10 需裝selenium 套件
(4)需要檔案: selenium_crawlerV4.py	:主程式
		chromedriver.exe		:瀏覽器驅動，如果你是用其他的瀏覽器可以自行替換
		parse.py			:用於分析輸入與檢查輸入，詳細規定參考下方，回傳一個參數列表
		input.txt			:輸入時間與路段，格式詳細規定請參考下方
		road.txt			:路段編號，是網站維護者台中市交通局定的，往後有新路段可以再加入

重要注意事項:

1.資料下載時間從2020至最晚為目前時間的上一個月。

2.input.txt格式: yyyy-mm yyyy-mm VXXXXXX VOOOOOO...
  前面兩項表示初始時間(含)結束時間(含)，初始與結束時間以空排分開，西元年與月以"-"分開。後面為路段編號可依需求給數個路段，
  以空白分開。路段編號可以到這網站查詢"http://e-traffic.taichung.gov.tw/RoadGrid/Pages/VD/History2.html"。
  假設要下載2021年1月到2022年7月，則輸入:2021-01 2022-07 VXXXXXX...

3.最小下載單位為一個月，資料下載格式是(.CSV)時間為每月1號的00:00到每月最後一天的23:00 
  e.g. 2020/1/01 00:00 - 2020/1/31 23:00

4.程式可能報錯原因:
  (1)時間格式錯誤(2020以前/不存在的時間/沒有使用上述格式/初始時間晚於結束時間...)
  (2)路段編號錯誤(如果至少存在一條正確路段，程式不會停止)
  (3)檔案開啟錯誤(不存在的檔案...)
  (4)除了可容許的路段錯誤外，其餘錯誤會返回空列表，在主程式exit()

5.該程式可能因為不同設備或網路環境而需要不同的設定，不然可能會有bug。原因可能在於當該程式模擬滑鼠
  點擊後，網頁回應沒有那麼快，需要用sleep函數等待一下，否則會有許多問題。在下載時間比較久遠的資料
  時，網頁載入也會較慢，需要調整來權衡效率與穩定性。大部分問題應該都可以藉由修改sleep函數來改善。

selenium_crawlerV4是基於selenium_crawlerV3的更新版本，以下是更新內容。
 (1)用輸入檔案來增加下載時間的彈性。
 (2)修復原先大小月計算出錯的bug。
 (3)增加可下載的檔案時間(最晚僅到目前時間的前一個月)。
 (4)檢查輸入時間與路段的合法性。
 (5)新增支援判斷閏年。
 (6)盡量精確錯誤回報。
